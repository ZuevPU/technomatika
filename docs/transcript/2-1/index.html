Конспект лекции · 2.1

Современная архитектура информационных технологий

Описание лекции

Лекция посвящена основам архитектуры IT‑проектов: их ключевым компонентам, ролям в команде, взаимодействию с другими областями знаний (биологией, финансами, медициной), а также современным тенденциям в разработке — использованию ИИ, различным технологиям и подходам к хранению данных.

Ключевые тезисы

1. Информатика как прикладная дисциплина

Информатика сама по себе не является самодостаточной областью — она всегда применяется в связке с другими предметами. Студент прикладной информатики, который разрабатывает мобильные приложения, работает на стыке технологий и конкретной предметной области. Информатика наиболее эффективна, когда интегрируется с другими сферами деятельности.

2. Компоненты IT‑проекта

IT‑проект состоит из четырёх ключевых элементов: данные, логика, интерфейс, инфраструктура. Данные — информация, собираемая датчиками или из таблиц; логика определяет работу проекта; интерфейс отвечает за внешний вид; инфраструктура — за платформу, на которой проект функционирует. Понимание этих компонентов необходимо для успешной разработки.

3. Роли в команде при разработке IT‑проекта

В команде разработки задействованы разные специалисты: дата‑инженер, аналитик данных, backend‑разработчик, front-end разработчик, DevOps, архитектор системы, тестировщик и менеджер проекта. Каждая роль отвечает за определённый аспект — от сбора и анализа данных до тестирования и управления процессом. Слаженная работа с чётким распределением ролей критически важна для успеха проекта.

4. Междисциплинарные связи IT

IT тесно переплетается с другими областями: биологией (расшифровка геномов), финансовыми технологиями (кредитный скоринг), медициной (анализ снимков МРТ и КТ). Сравнение геномов миллионов людей — задача класса Big Data. IT‑проекты могут решать сложные задачи на стыке разных дисциплин.

5. Роль искусственного интеллекта в разработке

На данный момент 75 % нового кода пишет искусственный интеллект, а вручную программисты создают лишь 5 % кода в новых стартапах. Платформы вроде вайп‑кода от Яндекса позволяют разрабатывать приложения с минимальным участием человека. ИИ существенно меняет подход к разработке IT‑проектов, снижая порог входа для непрограммистов.

6. Основные технологии веб‑разработки

Ключевыми технологиями для создания сайтов являются HTML (скелет сайта), CSS (стилизация) и JavaScript (поведение элементов). Современные фреймворки (React, Vue) и парадигма реактивности упрощают разработку интерактивных интерфейсов. Знание базовых технологий и современных фреймворков необходимо для веб‑разработки.

7. Подходы к хранению данных: SQL и NoSQL

Существуют два основных подхода: табличные базы данных (SQL) и нереляционные (NoSQL). SQL подходит для структурированных данных, но имеет сложности при работе с множеством пользователей; NoSQL эффективен для быстрого доступа к данным и используется в соцсетях и интернет‑магазинах. Выбор подхода зависит от специфики проекта.

8. Важность контроля версий и контейнеризации

Контроль версий помогает командам согласованно работать над проектом. Контейнеризация позволяет упаковывать приложение со всем окружением для безопасного переноса между системами. Эти инструменты повышают эффективность командной работы и безопасность проектов.

9. Цена ошибок в IT‑проектах

Ошибки в IT‑коде могут приводить к серьёзным последствиям и финансовым потерям. Ракета взорвалась через 37 секунд после старта из‑за переполнения переменной при пересчёте данных навигации — это обошлось в 370 миллионов. Тщательная проверка кода и внимание к деталям критически важны в IT‑разработке.

10. Принципы работы с ИИ в разработке

При использовании ИИ в разработке важно чётко понимать архитектуру проекта и ставить конкретные задачи. Необходимо проверять сгенерированный код, искать дефициты и использовать экспертную оценку. Работа с ИИ требует осознанного подхода и дополнительной проверки результатов.

Заключение

IT‑проекты — это междисциплинарные решения, объединяющие информатику с другими областями. Их успех зависит от продуманной архитектуры, слаженной работы команды и выбора подходящих технологий. Современные тенденции — активное использование ИИ, новые подходы к хранению данных и инструменты для командной работы — существенно меняют ландшафт IT‑разработки. При этом критически важно не забывать о качестве кода и цене возможных ошибок.

Рекомендации по изучению

Обратите внимание на взаимосвязь IT с другими дисциплинами и возможности межпредметных проектов. Осмыслите роли в IT‑команде и технологии, лежащие в основе веб‑разработки. Подумайте, как можно применять ИИ в образовательных проектах. Изучите базовые принципы работы с базами данных (SQL/NoSQL) и инструменты контроля версий. Попробуйте сгенерировать простой проект с помощью ИИ‑платформ и проанализируйте результат.
